## Context

Every list view in DocuFlow (and any consumer project) needs two
states that ln-acme does not currently ship:

1. **Skeleton** — what a loading table / card / list looks like before
   data arrives. Today consumers improvise with custom CSS.
2. **Empty state** — what a list looks like when it has zero data, or
   zero results after filtering. Two sub-types per `SESSION-HANDOFF.md`:
   "no data exists" vs "filter returned zero".

Both are in the claude-skills repo as specs (`empty-state.md`,
`loading-state.md`). This phase pulls the specs into concrete ln-acme
mixins + components.

## Pre-resolved ambiguities

### A. Skeleton API

Single primitive mixin: `@include skeleton` sets a gradient
background with `motion-safe` animation. Multiple shape helpers:

```scss
@include skeleton-text;   // single line
@include skeleton-text($lines: 3);   // multi-line paragraph
@include skeleton-circle($size: 2.5rem);   // avatars
@include skeleton-block($height: 8rem);    // cards, images
```

Default selector: `[data-ln-skeleton]`.

Usage:

```html
<div data-ln-skeleton>
	<div data-ln-skeleton-block style="height: 4rem"></div>  <!-- no -->
</div>
```

Correction: no inline styles. Consumer writes semantic selectors:

```scss
#folder-grid[data-loading] .card {
	@include skeleton-block;
}
```

Library ships a default `[data-ln-skeleton]` that gets `@include skeleton`.

### B. Skeleton animation

Shimmer gradient sweep, 1.5s linear infinite. Gated with `motion-safe`
(respects `prefers-reduced-motion` — falls back to a static gradient).

### C. Empty state API

Mixin: `@include empty-state`. Applied to `[data-ln-empty-state]` by
default. Structure:

```html
<div data-ln-empty-state>
	<svg class="ln-icon ln-icon--xl" aria-hidden="true">
		<use href="#ln-folder-off"></use>
	</svg>
	<h3>No documents yet</h3>
	<p>Upload your first document to get started.</p>
	<button type="button">Upload document</button>
</div>
```

Two sub-variants, selector-based (no BEM, no variant classes):

```scss
[data-ln-empty-state="no-data"] { /* optional override */ }
[data-ln-empty-state="no-results"] { /* optional override */ }
```

Default uses `centered column, gap 1rem, max-width 28rem, text-align
center`. The icon, h3, p, button are all optional — consumer picks
which to include.

### D. Selector semantics

- Default component selector: `[data-ln-skeleton]`, `[data-ln-empty-state]`.
- Mixins available for consumer semantic selectors:
  - `@include skeleton`, `skeleton-text`, `skeleton-circle`, `skeleton-block`
  - `@include empty-state`

### E. Commit strategy

Two commits per component (mixin, then component+docs+demo):

1. `feat(skeleton): add skeleton mixins`
2. `feat(skeleton): add [data-ln-skeleton] component + docs + demo`
3. `feat(empty-state): add empty-state mixin`
4. `feat(empty-state): add [data-ln-empty-state] component + docs + demo`

### Step 2 — Skeleton mixin (Commit 1)

Create `scss/config/mixins/_skeleton.scss`:

```scss
@use 'spacing' as *;
@use 'sizing' as *;
@use 'borders' as *;
@use 'motion' as *;

// Base skeleton — neutral gradient with shimmer sweep.
@mixin skeleton {
	background: linear-gradient(
		90deg,
		hsl(var(--color-neutral-200)) 0%,
		hsl(var(--color-neutral-100)) 50%,
		hsl(var(--color-neutral-200)) 100%
	);
	background-size: 200% 100%;
	@include rounded-sm;

	@include motion-safe {
		animation: ln-skeleton-shimmer 1.5s linear infinite;
	}
}

@keyframes ln-skeleton-shimmer {
	0%   { background-position: 200% 0; }
	100% { background-position: -200% 0; }
}

// Single line of skeleton text.
@mixin skeleton-text {
	@include skeleton;
	height: 0.875rem;
	width: 100%;
}

// Block (card, image placeholder).
@mixin skeleton-block($height: 6rem) {
	@include skeleton;
	height: $height;
	width: 100%;
	@include rounded-md;
}

// Avatar / icon placeholder.
@mixin skeleton-circle($size: 2.5rem) {
	@include skeleton;
	height: $size;
	width: $size;
	border-radius: 50%;
}
```

Register in `_index.scss`:

```scss
@forward 'loader';
@forward 'skeleton';
@forward 'tabs';
```

(Insert immediately after `loader` — loader and skeleton belong to the
same "waiting" family.)

# Skeleton

File: `scss/config/mixins/_skeleton.scss` + `scss/components/_skeleton.scss`.

Loader placeholder that appears while data is being fetched. Shimmer

## Mixins

| Mixin | Params | Output |
|---|---|---|
| `@include skeleton` | — | Base gradient + shimmer |
| `@include skeleton-text` | — | Single line, 14px tall |
| `@include skeleton-block` | `$height: 6rem` | Rectangular block |

| Attribute | Mixin applied |
|---|---|
| `data-ln-skeleton` | `skeleton` |
| `data-ln-skeleton-text` | `skeleton-text` |

## Usage

```html
<!-- Single line -->
<div data-ln-skeleton-text></div>

<!-- Card placeholder -->
<article data-ln-skeleton-block></article>

<!-- Avatar row -->

	<div data-ln-skeleton-circle></div>
	<div data-ln-skeleton-text></div>
</div>
```

## Project selectors

Use mixins on semantic selectors for custom skeleton shapes:

```scss
#document-card[data-loading] {
	.title { @include skeleton-text; }
	.meta  { @include skeleton-text; width: 60%; }
	.body  { @include skeleton-block; height: 8rem; }
}
```

## Motion safety

The shimmer animation is gated behind `@include motion-safe`. Users
with `prefers-reduced-motion: reduce` see a static gradient.

## Color

Gradient runs between `--color-neutral-200` and `--color-neutral-100`.
Works automatically in dark mode via the neutral scale inversion.

# Empty state

File: `scss/config/mixins/_empty-state.scss` + `scss/components/_empty-state.scss`.

Centered, icon-led placeholder for lists with no data.

## Two sub-types

1. **`no-data`** — list has never had items yet (onboarding moment).
   Show an inviting message and the primary "create" action.
2. **`no-results`** — list has items but the current filter/search
   returned zero. Show "no matches" messaging and a clear-filter
   action.

Distinguished by attribute value:

```html
<div data-ln-empty-state="no-data">
	<svg class="ln-icon ln-icon--xl" aria-hidden="true"><use href="#ln-folder"></use></svg>
	<h3>No documents yet</h3>
	<p>Upload your first document to get started.</p>
	<button type="button">Upload document</button>
</div>

<div data-ln-empty-state="no-results">
	<svg class="ln-icon ln-icon--xl" aria-hidden="true"><use href="#ln-search"></use></svg>
	<h3>No matches</h3>
	<p>Try a different search or clear your filters.</p>
	<button type="button">Clear filters</button>
</div>
```

## Structure

- **Icon** — `ln-icon--xl` (4rem), neutral-400 color
- **Title** — `h3`, typography role `heading-sm`
- **Description** — `p`, typography role `body-md`, secondary color
- **Action** — button or link, margin-top 0.5rem

All children optional. Consumer includes what makes sense.

## Default selector

`[data-ln-empty-state]` — applies `@include empty-state`. The attribute
value (`no-data` / `no-results` / any) is only for consumer CSS to
distinguish, if needed.

## Project selectors

```scss
#documents .empty { @include empty-state; }
```
```
