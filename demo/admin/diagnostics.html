    Diagnostics & DOM Linter

    Visual, CSS-based linter that runs only during local development. It flags integration mistakes, missing attributes, and DOM layout errors instantly in the browser without any runtime performance overhead.

    Development-Only Warning

    These warnings are powered by the ln-ashlar-dev.css stylesheet. In production environments, this stylesheet is excluded, guaranteeing zero layout overrides and zero performance penalty.

    1. Missing Sort Button Warning

    Flags a th[data-ln-table-sort] that is configured to be sortable but is missing the required <button data-ln-table-col-sort> trigger.

    Username Email
    petar_p petar@example.com
    marija_s marija@example.com

    2. Missing Table Wrapper ID Warning

    Flags interactive table wrappers [data-ln-table] that are missing a unique id attribute (which is required for mapping search and column filters correctly).

    Title Category
    Strategic Plan.pdf Planning

    3. Modal Trigger Missing Target Warning

    Flags a modal trigger [data-ln-modal-for] that is declared with an empty value, so it has no target modal id to open.

    4. Image Missing Alt Attribute

    Flags an <img> that has no alt attribute at all. An empty alt="" is intentionally allowed — it is the valid, WCAG-endorsed pattern for purely decorative images, so it is never flagged.

    5. Abbr Missing Title

    Flags an <abbr> that is missing its title attribute. Without a title the abbreviation has no expansion and is meaningless to readers and assistive tech.

    The agency known as NASA was founded in 1958.

    6. Empty Tooltip (No Title Fallback)

    Flags a [data-ln-tooltip] declared with an empty value and no native title fallback, so it would render an empty bubble. An empty data-ln-tooltip with a title present is valid (it falls back to the title) and is not flagged.

    7. Correct Implementation (No Errors)

    When the HTML structure and attributes are fully correct, no diagnostics warnings are shown.

    Writing Your Own Diagnostic Rules

    You can easily extend the linter by writing your own CSS selectors in scss/ln-ashlar-dev.scss and using the shared DRY mixins.

    The system provides two shared mixins in config/mixins/_diagnostics.scss:

    • @include dev-dom-error($message): Best for wrapping block elements with a dashed red border and a warning header.
    • @include dev-inline-error($message): Best for injecting red text warning labels next to small elements or headers.
    Example: Flagging form elements missing name attributes
    // Warns if any text input inside a validation form is missing its name attribute
[data-ln-validate] input[type="text"]:not([name]) {
    @include dev-dom-error("Input is missing a 'name' attribute, which prevents serialization.");
}
