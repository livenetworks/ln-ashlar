    Icons

    Overview & Philosophy

    Standard SVG icon strategies in modern web development present a difficult tradeoff: either developers must import massive icon library packages that bloat the production bundle, or manually copy-paste hundreds of raw, cluttered SVG path tags directly into their markup. Both approaches hurt the developer experience (DX) and slow down page load times.

    In ln-ashlar, this is solved by ln-icons, a zero-dependency **On-Demand SVG Sprite Generator**. Operating as a background service, it monitors the document for any vector reference matching the #ln- or #lnc- prefix, fetches the required icon asynchronously from a remote CDN, caches the response locally, and injects it into a single hidden SVG sprite sheet inside the DOM body.

    The Asset Pipeline Doctrine

    An icon utility is not responsible for defining or editing vector files. It acts strictly as an on-demand compiler. By mapping DOM references to distinct CDN endpoints, it ensures that your bundle size remains zero regardless of how many icons are loaded. The SVG `use` href attribute is the sole contract.

    Key Architectural Advantages

    1. Dual CDN Routing Channels: Automatically detects the prefix on the icon reference. #ln-{name} loads outline icons from the global Tabler Icons suite, while #lnc-{name} routes to a customizable corporate asset server for specialized brand icons.
    2. Local Caching Layer: Fetches each icon once. The raw vector payload is cached inside localStorage with a version prefix (lni:{name}). Subsequent page renders read the vector path directly from cache, resulting in instant rendering with zero network latency.
    3. Re-Rendering Observer: Equipped with a robust MutationObserver. Any dynamically created DOM node or runtime modification to an icon's href attribute (e.g. morphing a button icon to a checkmark) is captured and compiled instantly.

    For the full guide see the ln-icons README and the architecture reference.

    How it works

    Icons are fetched on demand from the Tabler Icons CDN (jsDelivr) and assembled into a hidden sprite in <body>. Any icon from Tabler works — just use its name as #ln-{name}. MutationObserver picks up icons added after page load automatically.

    Exception: #lnc-file-pdf, #lnc-file-doc, #lnc-file-epub are custom multi-color icons served from your own CDN — they do not follow currentColor.

    <!-- Standalone icon -->
<svg class="ln-icon" aria-hidden="true"><use href="#ln-home"></use></svg>

<!-- Icon in a button with text -->
<button>
    <svg class="ln-icon" aria-hidden="true"><use href="#ln-plus"></use></svg>
    Add User
</button>

<!-- Icon-only button — aria-label required -->
<button aria-label="Close">
    <svg class="ln-icon" aria-hidden="true"><use href="#ln-x"></use></svg>
</button>

<!-- Accordion chevron — CSS rotates on open -->
<header data-ln-toggle-for="panel1">
    Section Title
    <svg class="ln-icon ln-chevron" aria-hidden="true"><use href="#ln-arrow-down"></use></svg>
</header>

<!-- Custom icon (requires window.LN_ICONS_CUSTOM_CDN) -->
<svg class="ln-icon" aria-hidden="true"><use href="#lnc-file-pdf"></use></svg>

    Custom Icons (lnc-)

    Custom icons use the #lnc- prefix and are served from your own CDN. Set window.LN_ICONS_CUSTOM_CDN before the script loads:

    <script>
    window.LN_ICONS_CUSTOM_CDN = 'https://your-cdn.com/icons';
</script>
<script src="dist/ln-ashlar.iife.js" defer></script>

<!-- Use custom icons the same way -->
<svg class="ln-icon" aria-hidden="true"><use href="#lnc-my-icon"></use></svg>

    To add a new custom icon: create js/ln-icons/icons/{name}.svg with stroke="currentColor", run npm run build, then upload dist/icons/{name}.svg to your CDN.
