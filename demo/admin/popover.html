    Overview & Philosophy

    When presenting rich, structured, or interactive contextual information—such as user account actions, setting forms, help text, or nested detail lists—a standard tooltip is too restrictive. You need a modal-like dialog, but displayed inline relative to the trigger element.

    In ln-ashlar, this is managed by ln-popover. It is a highly-specialized JS component that coordinates click-triggered rich-content overlays. It is designed to be fully viewport-aware, ensuring overlays never spill outside the screen boundaries. It also handles automatic keyboard LIFO (Last-In, First-Out) ESC dismissal stacks and deep focus transitions out of the box.

    The Popover Doctrine

    The contract is absolute: the active state lives solely in the data-ln-popover attribute on the popover element, and the DOM is the single source of truth. Triggers, keyboard esc handlers, click-outside rules, and custom scripts all change state by writing to the attribute. A single MutationObserver callback runs the entire presentation, positioning, and focus management pipeline.

    Key Features & Viewport Awareness

    • Automatic Flipping (Auto-Flip): You can set a preferred position (data-ln-popover-position="top|bottom|left|right"). If there isn't enough room in the viewport, the positioning engine automatically flips the popover to the opposite side to keep it fully visible.
    • Focus Management: When a popover opens, focus moves automatically into the popover dialog. When closed, focus is returned back to the triggering element, maintaining a smooth accessibility path.
    • Dismissal Rules: Clicking anywhere outside the popover (or its trigger button) closes it. Additionally, pressing ESC closes the most recently opened popover in a LIFO stack.
    • Nesting & Composition: Opening popover B from inside popover A leaves A open. ESC will close B first, and then A.

    For the complete developer reference, see the ln-popover README and the architecture reference.
