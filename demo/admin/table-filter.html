    How It Works

    • Global search — ln-search dispatches ln-search:change events on the data-ln-table wrapper. ln-table intercepts the event and filters rows in-memory — no custom script needed.
    • Per-column filters — ln-filter dispatches ln-filter:changed events on the table wrapper. ln-table maps filter keys to columns via data-ln-filter-col on <th> elements. Multiple columns combine with AND logic; within a column, checked values use OR logic. When a filter is active, ln-table sets data-ln-filter-active on the <th> — the filter icon turns primary color.
    • Popover pattern — Department and Role use ln-popover with hardcoded checkboxes. Each popover's data-ln-filter targets the data-ln-table wrapper ID.
    • Dropdown pattern — Status uses ln-dropdown with hardcoded checkboxes embedded in the <th>.
    • Sort — ln-table-sort via data-ln-sort on <th> elements. Click header to cycle: ascending → descending → unsorted.

    Popover Pattern

    Filter trigger in <th>, popover with hardcoded checkboxes. The data-ln-filter targets the data-ln-table wrapper ID. The <th> needs data-ln-filter-col set to the filter key so ln-table can map it to the column index.

    <!-- Wrapper -->
<div data-ln-table id="my-table">
    <table>
        <thead>
            <tr>
                <th data-ln-sort="string">Name</th>
                <th data-ln-sort="string" data-ln-filter-col="dept">
                    Department
                    <button class="filter" type="button" data-ln-popover-for="filter-dept" aria-label="Filter department">
                        <svg class="ln-icon ln-icon--sm" aria-hidden="true"><use href="#ln-filter"></use></svg>
                    </button>
                </th>
            </tr>
        </thead>
        <tbody>...</tbody>
    </table>
</div>

<!-- Popover (teleported to body on open) -->
<div data-ln-popover id="filter-dept">
    <nav data-ln-filter="my-table">
        <label><input type="checkbox" data-ln-filter-key="dept" data-ln-filter-reset checked> All</label>
        <label><input type="checkbox" data-ln-filter-key="dept" data-ln-filter-value="IT"> IT</label>
        <label><input type="checkbox" data-ln-filter-key="dept" data-ln-filter-value="HR"> HR</label>
    </nav>
</div>

    Dropdown Pattern (Inline in TH)

    For columns with few known values. The dropdown and filter live inside the <th>. The <th> needs data-ln-filter-col set to the filter key.

    <th data-ln-sort="string" data-ln-filter-col="status">
    Status
    <div data-ln-dropdown>
        <button class="filter" type="button" data-ln-toggle-for="filter-status" aria-label="Filter status">
            <svg class="ln-icon ln-icon--sm" aria-hidden="true"><use href="#ln-filter"></use></svg>
        </button>
        <ul id="filter-status" data-ln-toggle>
            <li>
                <nav data-ln-filter="my-table">
                    <label><input type="checkbox" data-ln-filter-key="status" data-ln-filter-reset checked> All</label>
                    <label><input type="checkbox" data-ln-filter-key="status" data-ln-filter-value="Active"> Active</label>
                    <label><input type="checkbox" data-ln-filter-key="status" data-ln-filter-value="Inactive"> Inactive</label>
                </nav>
            </li>
        </ul>
    </div>
</th>

    Search Integration

    ln-search dispatches ln-search:change on the target element. When the target is a data-ln-table wrapper, ln-table intercepts the event, calls preventDefault(), and handles filtering in-memory. No custom script needed.

    <!-- Search input targets the ln-table wrapper -->
<input type="search" placeholder="Search..." data-ln-search="my-table">

<!-- ln-table wrapper -->
<div data-ln-table id="my-table">
    <table>...</table>
</div>
