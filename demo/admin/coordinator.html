    Data Coordinator

    Overview & Decoupled Architecture

    The Data Coordinator (data-ln-data-coordinator) acts as the parent orchestrator of the decoupled 3-Tier Local-First architecture in ln-ashlar. It encapsulates two completely blind sibling concerns—local database cache storage and transport networking gateway—and wires them together using event-driven bubbling and Promise chains.

    The Orchestrator Doctrine

    The Cache Store (data-ln-data-store) has zero network logic and only fires request-remote-* events. The Connector (data-ln-api-connector) only makes fetch requests. The parent Data Coordinator listens for the store's requests, applies bidirectional Ingress/Egress data schema restructuring, delegates requests to the connector, and confirms mutations back in the database.

    Decoupled Setup Advantages

    • Zero-Coupling Code: You can completely swap the transport network layer (e.g. from REST API to CouchDB or WebSockets) without touching a single line of your database cache, forms, or data table markup.
    • Secure JS Mappers: Register mappers securely via window.lnCore.registerDataMapper() to map, flatten, or transform server schemas to clean local database schemas on the fly, keeping markup 100% CSP-compliant.
