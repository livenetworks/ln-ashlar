    Tooltip

    Overview & Philosophy

    Sighted users require brief, high-impact labels when interacting with icon-only buttons or complex technical abbreviations. However, loading heavy JavaScript positioning libraries for simple tooltips introduces unnecessary runtime overhead, while using standard native browser title attributes results in laggy, non-stylable, and uncooperative tooltips.

    In ln-ashlar, this is solved by ln-tooltip, a zero-dependency Dual-Layer Tooltip Primitive. It is engineered on a progressive enhancement architectural model. By default, it ships a pure CSS baseline (zero JS footprint) that styles and positions a tooltip bubble using standard ::after pseudo-elements. For advanced requirements, developers can opt into the JS enhancement layer to enable viewport-aware auto-flipping, text wrapping, and clean accessibility portals.

    The progressive Tooltip Doctrine

    A tooltip is a non-interactive decorative bubble. For interactive panels, menus, or click-triggered overlays, use ln-popover. The CSS baseline handles standard layouts cleanly. The JS portal layer should only be used when escaping ancestor clipping or requiring screen reader descriptions. The `data-ln-tooltip` attribute is the sole contract.

    Architectural Design Features

    1. CSS Baseline (Pure CSS, Zero JS): Triggers instantly on hover or keyboard focus using standard pseudo-elements. Super-lightweight, zero bundle size impact.
    2. JS Progressive Portal Layer: Appends the tooltip to a global #ln-tooltip-portal at the body root, allowing tooltips to escape parent containers styled with overflow: hidden.
    3. Viewport-Aware Placement: Preferred positioning (top, bottom, left, right) serves as a starting seed. If the bubble hits the edge of the viewport, the JS layer automatically flips it to the opposite side.
    4. Accessibility Integration: Automatically wires aria-describedby on the active focus target, feeding the tooltip content directly into the browser's accessibility tree for screen readers.

    For the full guide see the ln-tooltip README and the architecture reference.

    7. JS enhance — keyboard focus + aria-describedby

    Instructions: Tab to the button below. Open DevTools (F12) and switch to the Elements panel. Inspect the button — it should have aria-describedby="ln-tooltip-N". Inspect #ln-tooltip-portal in the body — there should be a <div class="ln-tooltip"> with the matching id. Tab away — both attributes should disappear.

    8. JS enhance — auto-attach on [data-ln-tooltip][title]

    Any element with both data-ln-tooltip and title auto-enhances — the JS layer attaches without requiring data-ln-tooltip-enhance. The CSS baseline cannot strip title, so the browser's native tooltip would leak alongside the styled one. Auto-enhance resolves this for the semantic <abbr> pattern.

    Hover (or tab to) ISO. Open DevTools and watch the title attribute — it is stripped while the styled tooltip is visible and restored on hide. No data-ln-tooltip-enhance flag is required.
