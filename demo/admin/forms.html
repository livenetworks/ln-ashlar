    Forms & Buttons

    CSS Grid + <div class="form-element"> pattern. No <div class="form-group"> or <div class="form-row">. The grid controls the layout via SCSS — not via HTML.

    Principle

    Each field is a <div class="form-element"> containing a <label for="..."> and <input id="..."> (explicit association). The root <form> gets @include form-grid in SCSS — a 6-column grid, collapsing to 1 column on mobile. Column widths are set via nth-child in form-specific SCSS.

    Wrong — wrapping label

    <!-- implicit label, no for/id -->
<label>
  Name
  <input type="text" name="name">
</label>

<label>
  City
  <input type="text" name="city">
</label>

    Correct — form-element

    <!-- explicit for/id, div wrapper -->
<form id="my-form" data-ln-form>

  <div class="form-element">
    <label for="name">Name</label>
    <input type="text" id="name" name="name"
           required data-ln-validate>
    <ul data-ln-validate-errors>
      <li hidden data-ln-validate-error="required">
        Required field
      </li>
    </ul>
  </div>

</form>
    Element Rule
    Root <form id="..." data-ln-form> — styled with @include form-grid in SCSS (6 cols → 1 col on mobile)
    Children <div class="form-element"> — wraps <label> + <input> + error list
    Label Explicit for/id association: <label for="name"> + <input id="name">
    Validation data-ln-validate on input + <ul data-ln-validate-errors> with <li data-ln-validate-error="...">
    Column spans In form-specific SCSS via .form-element:nth-child(N) — NEVER width classes on the element
    Pill radio/checkbox <ul> > <li> > <label> > <input> — grouped pills (see Pill Labels section)
    .form-actions <ul class="form-actions"> with <li> per button — spans full-width in SCSS

    Fill — edit mode
    
					
    
						
    Populate form fields programmatically. Two patterns:
    
						
    
							
							
							
						
    
						
    Code
    
						
    // Pattern 1: Direct API — caller has a reference to the form
form.lnForm.fill({ fname: 'Dalibor', email: 'd@test.com' });

// Pattern 2: Coordinator — routes external events to the form
// Table doesn't know about the form. Form doesn't know about the table.
// The coordinator is the glue.
table.addEventListener('ln-data-table:row-click', function (e) {
    form.lnForm.fill(e.detail.record);
});

// Reset
form.lnForm.reset();

    Buttons — Architecture
    
					
    
						
    Three tiers. No class needed for the first two — the browser type attribute determines the role.
    

						
    1. Neutral — <button type="button">
    
						
    All buttons get structure (padding, font, flex, hover) from _global.scss. Cancel/Close/Toggle buttons need no class.
    
						
    
							
							
							
						
    

						
    2. Primary — type="submit"
    
						
    Submit buttons automatically receive primary fill color. Structure inherited from tier 1. No class needed.
    
						
    
							
							
						
    

						
    3. Action button — class="btn"
    
						
    Non-submit action buttons (Add, Delete, Export). In production, apply via @include btn on a semantic selector in project SCSS. class="btn" is for prototyping only.
    
						
    
							
							
							
							
							
							
						
    

						
    Color change — override --color-primary
    
						
    Set --color-primary on the button or any parent. All states (hover, active, focus-ring) adapt automatically.
    
						
    /* Project SCSS — semantic selector */
#delete-user  { @include btn; --color-primary: var(--color-error); }
#approve-user { @include btn; --color-primary: var(--color-success); }

/* Context override — all submit + action buttons inside change */
.danger-zone  { --color-primary: var(--color-error); }
    
					
    
				
    

				
				
    
					
    Button Sizes
    
					
    
						
    Size modifiers: class="btn btn-sm" and class="btn btn-lg". In projects, apply via @include btn-sm / @include btn-lg after @include btn.
    
						
    
							
							
							
						
    
						
    
							
							
							
						
    
					
    
				
    

				
				
    
					
    Icon-only Buttons
    
					
    
						
    Icon-only buttons use no class — they get neutral global styling (transparent, gray hover). No padding override needed.
    
						
    
							
							
							
							
							
						
    
					
    
				
    

				
				
    
					
    Button Groups — btn-group
    
					
    
						
    Row action buttons grouped with small gap. In projects: #users-table .row-actions { @include btn-group; }

  <div class="form-element">
    <label for="email">Email</label>
    <input type="email" id="email" name="email" required data-ln-validate>
    <ul data-ln-validate-errors>
      <li hidden data-ln-validate-error="required">Required</li>
      <li hidden data-ln-validate-error="typeMismatch">Invalid email</li>
    </ul>
  </p>

  <div class="form-element">
    <label for="category">Category</label>
    <select id="category" name="category">
      <option value="a">Option A</option>
    </select>
  </p>

  <ul class="form-actions">
    <li><button type="button">Cancel</button></li>
    <li><button type="submit">Save</button></li>
  </ul>

</form>
    

						
    SCSS
    
						
    #my-form {
  @include form-grid;

  .form-element { grid-column: span 3; }                   // default: half
  .form-element:nth-child(3) { grid-column: span 6; }     // category: full
  > fieldset { grid-column: span 6; }
  .form-actions { grid-column: span 6; }
}
    

						
    Coordinator
    
						
    document.addEventListener('ln-form:submit', function (e) {
    if (e.target.id !== 'my-form') return;

    fetch('/api/resource', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(e.detail.data)
    })
    .then(function (res) { return res.json(); })
    .then(function () {
        window.lnToast.enqueue({ type: 'success', title: 'Saved' });
    });
});
