Translations

Overview & Philosophy

Building forms that support multi-lingual input editing presents massive layout and data organization challenges. Standard approaches either require developers to render side-by-side tabs for every single field—bloating the DOM and confusing the user—or build heavy client-side templates that serialize and parse nested JSON fields manually before sending requests to the server.

In ln-ashlar, this is solved by ln-translations, a zero-dependency **Form Translation Coordinator**. Operating as a form-level supervisor, it intercepts language selection events, manages active language list badges, and dynamically duplicates translatable fields inside the DOM. It automatically handles input cloning, coordinates custom menus, auto-detects pre-rendered server payloads, and generates structured name attributes ready for backend database mapping.

The Translation Mapping Doctrine

A translation utility should only coordinate input fields and serialization layouts. It remains detached from language dictionaries or API translating services. It operates on structured form boundaries, mapping fields dynamically using deterministic HTML attributes. The generated input name attribute is the sole contract.

Key Architectural Advantages

  1. On-Demand Field Cloning: Translatable containers carrying data-ln-translatable serve as templates. When a language is toggled, the coordinator clones the internal input or textarea, sets appropriate placeholder guides, and mounts it cleanly.
  2. Clean Serialization Prefixes: Supports simple fields and complex nested lists. Cloned inputs compile structured array names (e.g. trans[en][title] or items[1][trans][en][title]), allowing standard form submissions to parse arrays natively.
  3. Server & UI Coordination: Detects pre-rendered localized fields on load seamlessly, integrates with `ln-dropdown` to manage locale selection menus, and renders active language badges using standard HTML <template> elements.

For the full guide see the ln-translations README and the architecture reference.
