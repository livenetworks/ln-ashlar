Time

Timezone-aware timestamp formatting using Intl.DateTimeFormat and Intl.RelativeTimeFormat. Enhances <time> elements — Blade renders fallback text, JS replaces with localized output.

Attributes

AttributeOnDescription
data-ln-time="mode"<time>Format mode: relative, short, full, date, time
datetime="1736952600"<time>Unix timestamp in seconds
data-ln-time-locale="mk"<time>Override locale (default: <html lang> or browser)

Format Modes

ModeExample OutputIntl Options
relative3 hr. ago, just now, in 2 hr.Intl.RelativeTimeFormat
shortJan 15 / Jan 15, 2024{ month: 'short', day: 'numeric' } + conditional year
fullJanuary 15, 2025 at 2:30 PM{ dateStyle: 'long', timeStyle: 'short' }
dateJan 15, 2025{ dateStyle: 'medium' }
time2:30 PM{ timeStyle: 'short' }

Edge Cases

CaseElementResult
Empty datetime fallback text Skipped — fallback text preserved
Missing datetime no datetime attr Skipped — text preserved
Non-numeric datetime ISO string Skipped — not a Unix timestamp
Epoch zero epoch Rendered normally (Jan 1, 1970)

HTML Structure

<!-- Blade renders fallback, JS enhances -->
<time data-ln-time="relative" datetime="1736952600">Jan 15</time>
<time data-ln-time="short" datetime="1736952600">Jan 15</time>
<time data-ln-time="full" datetime="1736952600">January 15, 2025</time>
<time data-ln-time="date" datetime="1736952600">Jan 15, 2025</time>
<time data-ln-time="time" datetime="1736952600">14:30</time>

<!-- Locale override -->
<time data-ln-time="full" datetime="1736952600" data-ln-time-locale="mk">fallback</time>

API

// Manual init (not needed for normal use — MutationObserver handles it)
window.lnTime(document.body);

// Instance API
const el = document.querySelector('time[data-ln-time]');
el.lnTime.render();    // Force re-render
el.lnTime.destroy();   // Cleanup (remove from auto-update pool)

// Reactive update — just change the attribute
el.setAttribute('datetime', '1700000000');
el.setAttribute('data-ln-time', 'date');  // switch mode
