Time

Overview & Philosophy

Displaying dates, times, and relative periods (e.g. "3 hours ago") in multi-regional web portals is surprisingly complex. Server-side rendering often binds timestamps to a single static timezone, leaving international users to do timezone math in their heads. At the same time, loading massive, multi-kilobyte datetime formatting libraries in the browser compromises initial page loading speeds.

In ln-ashlar, this is solved by ln-time, a zero-dependency **Timezone-Aware Timestamp Formatter**. Operating as a progressive enhancement layer, it upgrades native HTML <time> elements. Server engines render a clean UTC date or relative string fallback. On script boot, the component parses the Unix timestamp, formats the output based on the user's local timezone and browser language using high-performance standard APIs, and schedules a single background interval to update relative timestamps live.

The Localized Datetime Doctrine

A time formatter does not parse complex custom string tokens or manage calendars. It acts strictly as an enhancer of standard HTML `` tags. It offloads localized translations and timezone offsets directly to standard browser APIs. The native `datetime` attribute containing a Unix timestamp in seconds is the sole contract.

Key Architectural Advantages

  1. Standard-Based Concurrency (Zero Bloat): Offloads formatting entirely to native Intl.DateTimeFormat and Intl.RelativeTimeFormat APIs. Bypasses heavy parsing packages to maintain a zero-bloat footprint.
  2. Centralized Interval Timer: Relative timestamps are updated dynamically. To maximize performance, the component runs a single global 60-second background ticker to update all elements on the page, automatically sweeping away orphaned nodes.
  3. Reactive Property Mutation: Fully integrates with modern reactive workflows. Any dynamic updates to datetime or data-ln-time attributes via JavaScript are observed and rendered instantly.

For the full guide see the ln-time README and the architecture reference.

API

// Manual init (not needed for normal use — MutationObserver handles it)
window.lnTime(document.body);

// Instance API
const el = document.querySelector('time[data-ln-time]');
el.lnTime.render();    // Force re-render
el.lnTime.destroy();   // Cleanup (remove from auto-update pool)

// Reactive update — just change the attribute
el.setAttribute('datetime', '1700000000');
el.setAttribute('data-ln-time', 'date');  // switch mode
