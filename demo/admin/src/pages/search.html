Search

Generic search component — filters children of a target element by textContent. Works on lists, card grids, or any container. Also integrates with ln-table for in-memory table filtering.

Attributes

AttributeOnDescription
data-ln-search="targetId"wrapper or <input>Target element by ID
data-ln-search-hidechildren of targetSet by JS when element doesn't match

On a wrapper: finds input[type="search"], input[type="text"], or [name="search"] inside.
Directly on <input>: that input is used.

Events

EventOnCancelabledetail
ln-search:changetarget elementyes{ term, targetId }

Fires on the target element (not the input). If preventDefault() is called, ln-search skips its default show/hide — the consumer handles filtering (e.g. ln-table).

Table Integration

When data-ln-search points to a [data-ln-table] element, ln-table intercepts ln-search:change and calls preventDefault(). No extra configuration — the integration is automatic.

<input type="search" data-ln-search="my-table">

<div id="my-table" data-ln-table>
    <table>...</table>
</div>

See the Data Table and Table demos for live examples.

Combination with ln-filter

ln-search and ln-filter work independently on the same target. Each uses its own hide attribute. An element is visible only when no hide attribute is present:

[data-ln-search-hide],
[data-ln-filter-hide] {
    display: none;
}

HTML Structure

<!-- On a wrapper -->
<label data-ln-search="my-list">
    <span class="ln-icon-search ln-icon--sm"></span>
    <input type="search" placeholder="Search...">
</label>

<ul id="my-list">
    <li>First element</li>
    <li>Second element</li>
</ul>

<!-- Directly on input -->
<input type="search" data-ln-search="my-list">

Required CSS

/* Hide non-matching elements */
[data-ln-search-hide] {
    display: none;
}
