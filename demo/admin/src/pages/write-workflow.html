Overview — the form as the universal write-entry

This page demonstrates the declarative form → data-layer write pipeline: a single <form data-ln-form data-ln-form-scope> is the only write-entry point. No application JavaScript builds ln-store:request-create/request-update events by hand — a valid submit is left native, and ln-data-coordinator claims it (preventDefault() on a document-level submit listener), serializing it itself; here the form carries a named scope (data-ln-form-scope="wwf-documents") that is matched against the coordinator by name rather than DOM containment, interprets the effective HTTP method (POST → create, PUT/PATCH → update), and routes it through the existing store → connector pipeline. An empty data-ln-form-scope would instead fall back to nearest-ancestor containment.

The scoped form still carries a real action + method="post" and the data-ln-form-action-edit fallback — without JavaScript, the exact same markup degrades to a native HTML submit.
