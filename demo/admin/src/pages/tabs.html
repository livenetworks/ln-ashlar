Overview

N-way exclusive panel selection on a single container — the tab primitive in ln-ashlar. 145 lines of JS that own three things at once: which tab is active (a string key on the wrapper), how to persist it (URL hash for namespaced sections, localStorage as the opt-in fallback), and how to keep ARIA in sync.

The contract is one sentence: the active tab lives in data-ln-tabs-active on the wrapper, and the wrapper is the source of truth. Click handlers, hash sync, persistence restore, external code — every entry point funnels into one setAttribute('data-ln-tabs-active', key) call. A single MutationObserver callback runs the activation pipeline. There is no second path.

Tabs are deliberately NOT built on ln-toggle. The contracts diverge: ln-toggle is binary open/close, tabs are N-way exclusive; toggle uses aria-expanded (disclosure), tabs use aria-selected (tablist); toggle persists per element, tabs use namespace-scoped URL hashes OR per-page localStorage (mutually exclusive). Squeezing tabs into N coordinated toggles would put coordinator logic in the wrong place. See the README "Why not ln-toggle" section for the full architectural argument.

For the full guide see the ln-tabs README and the architecture reference.

Anatomy

A tabs section has three structural pieces — a wrapper, a list of tab buttons, and a list of matching panels. The pairing is by key, not by DOM position.

<section id="user-tabs" data-ln-tabs data-ln-tabs-default="info">
    <nav>
        <button data-ln-tab="info">Information</button>
        <button data-ln-tab="settings">Settings</button>
    </nav>

    <section data-ln-panel="info">…</section>
    <section data-ln-panel="settings" class="hidden">…</section>
</section>

The wrapper — [data-ln-tabs]

The element with data-ln-tabs is the tabs root. Presence of the attribute creates the instance; the value is unused. The wrapper hosts the active-key attribute (data-ln-tabs-active, written by the component), the default-key attribute (data-ln-tabs-default), and the namespace (id or data-ln-tabs-key). Use <section>, <div>, <article>; do NOT use <main>.

The tab buttons — [data-ln-tab="key"]

Each tab button is queried by a deep selector inside the wrapper. The buttons are typically inside a <nav> for grouping, but the position is not enforced. Each button SHOULD be a <button> — the global library style and the tabs-tab mixin assume it. Inside a form, every tab button needs type="button" to avoid submitting the form on click.

The panels — [data-ln-panel="key"]

Each panel matches a tab by key. Inactive panels MUST be shipped with class="hidden" so they start hidden — without it, the user sees a flash of all-panels-stacked before the component initializes and collapses to one.

Triggers may be <button> or <a href="#nsKey:key">. Anchor triggers produce shareable URLs (right-click → copy link, middle-click → open in new tab) — the click handler calls event.preventDefault() so the browser's default navigation does not race with the JS state write. Use the boolean form (data-ln-tab with no value) on anchors so the key only appears in href; explicit data-ln-tab="key" works too if you prefer the redundancy. See the live demo below.

What state goes where

"Which tab is active?" lives in data-ln-tabs-active on the wrapper. "Is this tab visually selected?" lives in data-active + aria-selected on the button (synced from the wrapper's attribute). "Is this panel visible?" lives in the .hidden class + aria-hidden on the panel (also synced). The component instance carries caches (mapTabs, mapPanels) and configuration (defaultKey, nsKey, autoFocus), but every observable piece of state is on the DOM.

Single source of truth

Every consumer that wants to change the active tab writes data-ln-tabs-active on the wrapper. There is no second path — click handlers and the hashchange listener both end at setAttribute.

const tabs = document.getElementById('user-tabs');

// Canonical write — switches the active tab
tabs.setAttribute('data-ln-tabs-active', 'settings');

// Via hash (when hash sync is enabled)
location.hash = 'user-tabs:settings';
location.hash = 'user-tabs:settings&project-tabs:members';

All paths produce the same data-active / aria-selected flips on tab buttons, the same .hidden / aria-hidden toggles on panels, the same ln-tabs:change event, the same localStorage write (if persistence is opted in and hash sync is off).

API

The component does not generate the href for you — the markup author writes it. The format mirrors the click-driven hash exactly.

The click handler calls event.preventDefault() so the browser's default navigation does not race with the JS state write. Ctrl/Meta-click and middle-click skip preventDefault and bubble to the browser as expected — the standard "open in new tab" affordance.

Common mistakes

1. Tab buttons inside a form without type="button"

The HTML default for <button> is type="submit". A tab click inside a form will submit the form before the tab handler runs. Every tab button inside a form needs type="button".

2. Forgetting class="hidden" on inactive panels

Until the component initializes, all panels are visible. The user sees a flash of all-panels-stacked before they collapse to one. Ship inactive panels with class="hidden".

3. Mixing hash sync and data-ln-persist

The two persistence mechanisms are mutually exclusive. id (or data-ln-tabs-key) enables hash sync, which silently disables data-ln-persist. Pick one.

4. Programmatically adding tabs after init

The instance's mapTabs / mapPanels are populated at _init — adding new [data-ln-tab] / [data-ln-panel] children later does NOT register them. To add tabs dynamically: el.lnTabs.destroy() + modify DOM + window.lnTabs(parent), or replace the entire wrapper subtree (the document-level observer auto-inits a new wrapper).

5. Expecting keyboard arrow nav

ln-tabs does not implement Arrow Left/Right or Home/End. Tab key cycles through tab buttons via the browser default; Enter/Space activates. Arrow nav is a WAI-ARIA enhancement, not baseline. Wire it on the consumer side if your audit requires it.

6. Putting data-ln-tabs on <main>

HTML spec allows only one <main> per page. Use <section>, <aside>, <article>, or <div>.

HTML

<!-- Namespace = id attribute (or data-ln-tabs-key) -->
<section id="my-tabs" data-ln-tabs data-ln-tabs-default="first">

    <!-- Tab buttons — type="button" if inside a form! -->
    <nav>
        <button type="button" data-ln-tab="first">First tab</button>
        <button type="button" data-ln-tab="second">Second tab</button>
    </nav>

    <!-- Panels — inactive ones start hidden -->
    <section data-ln-panel="first">
        Content of the first tab
    </section>

    <section data-ln-panel="second" class="hidden">
        Content of the second tab
    </section>

</section>
