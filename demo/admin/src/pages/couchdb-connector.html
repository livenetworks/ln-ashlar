Overview & Architecture

In the decoupled 3-tier data lifecycle of ln-ashlar, the Transport Gateway (data-ln-couchdb-connector) owns all CouchDB credentials, database paths, auth hashes, and headers. It is completely blind to database caches and schemas, communicating solely via CustomEvent channels or JS-level promise APIs. It implements the CouchDB changes feed and bulk documents protocols seamlessly.

Dynamic CouchDB Synchronization

Mutating data-ln-couchdb-url, data-ln-couchdb-db, data-ln-couchdb-auth, or custom headers directly on the DOM element at runtime will instantly re-evaluate the instance configuration via dynamic MutationObservers and dispatch ln-couchdb-connector:config-changed.
