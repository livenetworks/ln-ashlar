Overview & Philosophy

When an asynchronous event completes, or a background system action finishes—such as saving a record, encountering a network error, or warning the user about a quota threshold—we need to notify the user. However, a modal dialog is too disruptive. We need a non-blocking, temporary notification card that slides into view at the corner of the screen.

In ln-ashlar, this is managed by ln-toast. It is a service-style, zero-markup component that handles a dynamic queue of alert cards. It exposes a single, global event listener, builds notification cards from templates dynamically, manages auto-dismiss timers, supports rich layouts (including bulleted validation error maps), and cleans up DOM elements after exit transitions.

The Toast Doctrine

Toasts are completely decoupled from UI producers. Components never import or call toast APIs directly. To trigger a notification, any script or component anywhere in the page simply dispatches an ln-toast:enqueue event on the window object. The central toast coordinator catches the event, adds it to the rendering queue, and handles the DOM. The window event is the sole contract.

Dynamic Features & Sizing

  • Automatic Dismissal: Every toast card automatically triggers its close animation after 6 seconds. If a user hovers over a toast, the timer is paused, resuming once the mouse leaves.
  • Validation Lists: If the message detail is an array of strings, the toast automatically renders it as an unordered list, perfect for displaying multiple form validation errors in a single compact card.
  • Categorized Types: Supports success (green accent), error (red accent), warn (yellow accent), and info (blue accent) with matching icon sprites dynamically loaded.

For the complete developer reference, see the ln-toast README and the architecture reference.

Examples

To display a notification programmatically, dispatch a custom event to the window object: window.dispatchEvent(new CustomEvent('ln-toast:enqueue', { detail: { type: 'success', title: 'Title', message: 'Message' } })).

Error with a list of errors

Toasts support an array of strings in message to render a bulleted list of errors: message: ['Error 1', 'Error 2'].
