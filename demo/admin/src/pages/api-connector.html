Overview & Architecture

In the decoupled 3-tier data lifecycle of ln-ashlar, the Transport Gateway (data-ln-api-connector) owns all credentials, base URLs, resource paths, and headers. It is completely blind to database caches and schemas. UI widgets and Cache stores talk to it solely via CustomEvent channels or JS-level promise APIs.

Dynamic Configuration Synchronization

Mutating data-ln-api-base-url, data-ln-api-path, or data-ln-api-headers directly on the DOM element at runtime will instantly re-evaluate the instance configuration and dispatch ln-api-connector:config-changed.
