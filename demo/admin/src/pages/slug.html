ln-slug

Auto-slug micro-component — mirrors a slugified version of a source field into a slug field while the slug is pristine. Stops when the user types directly into the slug field; resumes if cleared.

Overview

Add data-ln-slug-from="sourceName" to a slug <input> inside a <form>. The component watches the source field and mirrors slugify(source.value) into the slug field as long as it is pristine (empty or not yet edited by the user).

AttributeOn elementDescription
data-ln-slug-from="name"<input>Activates slug mirroring from the named source field in the same form

Example — Live mirroring (empty slug = pristine)

Type in the Name field. The Slug updates automatically. Type directly into Slug to stop mirroring; clear it to resume.

Example — Pre-filled slug is NOT overwritten

When the slug field already has a value at page load (e.g. from a server-rendered edit form), it starts NON-pristine. Typing in the name field does NOT overwrite the existing slug.

Pristine rules

SituationResult
Slug field empty at initPristine — mirroring active
Slug field non-empty at init (server-rendered)Not pristine — never overwrite
User types non-empty into slugNot pristine — mirroring stops
User clears slug to emptyPristine — mirroring resumes
lnForm.fill sets non-empty slugNot pristine — loaded slug preserved
Form reset clears slug to emptyPristine — mirroring resumes

Limitations (v1)

  • ASCII-only: Cyrillic, Macedonian, accented characters, and other non-ASCII input are stripped entirely (become -). No transliteration in v1.
  • Source read at init: The source field is resolved once at construction. Dynamically added source fields require re-initialization.
  • Single source field: If multiple fields share the same name (RadioNodeList), the component warns and bails. One text input → one slug.
