Sortable

Overview & Philosophy

Drag-and-drop interfaces in modern web applications are notoriously difficult to implement correctly. Standard HTML5 Drag & Drop APIs are clunky, inconsistent across browsers, and fail completely on touch devices without third-party polyfills. Conversely, heavy UI frameworks require importing megabytes of dependencies just to reorder items inside a list.

In ln-ashlar, this is solved by ln-sortable, a zero-dependency **Drag & Drop Reordering Primitive**. Built natively on the browser's Pointer Events API, it supports both touch and mouse operations out of the box. It separates visual DOM layout changes from business logic: focusing strictly on re-sorting elements in-place and dispatching high-impact telemetry events so that parent coordinators can synchronize data models or issue server updates.

The Reordering Doctrine

A sortable primitive is not responsible for updating databases, syncing state stores, or managing layout transitions. It acts as an isolated DOM manipulator. It provides layout shifts and dispatches index changes, leaving styling to standard CSS classes. The HTML container attribute is the sole contract.

Architectural Design Features

  1. Pointer API Concurrency: Unified mouse and touch track tracking. The component locks screen scrolling on drag handles using touch-action: none to prevent drag delays or stuttering on mobile screens.
  2. Attribute Governance: Drag status is managed entirely by the data-ln-sortable attribute on the parent list. Setting it to "disabled" deactivates sorting instantly, which is perfect during network saves or locked layouts.
  3. Clean Telemetry Hooks: Emits clean cancelable events (ln-sortable:before-drag) to prevent accidental drags, and completion notifications (ln-sortable:reordered) detailing the exact oldIndex and newIndex shifts for instant store updates.

For the full guide see the ln-sortable README and the architecture reference.

    API

    var list = document.querySelector('[data-ln-sortable]');

// Via API (sets the attribute, observer syncs state)
list.lnSortable.enable();     // sets data-ln-sortable=""
list.lnSortable.disable();    // sets data-ln-sortable="disabled"
list.lnSortable.isEnabled;    // boolean

// Via attribute (identical result)
list.setAttribute('data-ln-sortable', 'disabled');
list.setAttribute('data-ln-sortable', '');

// Constructor — only for non-standard cases
window.lnSortable(container);
