Overview

The smallest reactive primitive in ln-ashlar: an attribute on an element, a class derived from that attribute, and a pair of events on each side of the transition. 145 lines of JS that exist so every other open/close component in the library — accordion, dropdown, sidebar drawer, dismissible alert, language menu — does not have to re-implement the same state machine.

The contract is one sentence: the attribute is the state, and the state is the attribute. data-ln-toggle="open" means open; anything else ("close" or empty) means closed. API methods, trigger clicks, sibling components, external scripts, and DevTools edits all funnel into one setAttribute call. A single MutationObserver callback runs the open/close pipeline. There is no second path.

For the full guide see the ln-toggle README and the architecture reference. Read it before reaching for ln-accordion or ln-dropdown — both are thin layers on top of the toggle's contract.

Anatomy

A toggle has two halves: the panel (the thing that opens and closes) and the trigger (the thing that controls it). The pairing is by ID; the elements can live anywhere in the DOM relative to each other.

<!-- Trigger anywhere -->
<button data-ln-toggle-for="my-panel">Toggle</button>

<!-- Panel anywhere -->
<section id="my-panel" data-ln-toggle class="collapsible">
    <article class="collapsible-body"><p>Hello.</p></article>
</section>

The panel — [data-ln-toggle]

The presence of the attribute is what makes the element a toggle; the value is the state. "open" is open; anything else is closed. Empty (data-ln-toggle with no value) is also closed — the comparison is literal-string equality with "open".

The element should have an id so triggers can reference it. The element type is the consumer's choice — <section>, <aside>, <article>, <div> all work. Do NOT use <main> — the HTML spec allows only one per page.

The trigger — [data-ln-toggle-for="id"]

Any element with data-ln-toggle-for becomes a click target. <button> is the right semantic for a state-toggling action; <header> is also common in accordion patterns where the entire heading is the clickable surface. The pairing is purely by ID — a trigger in the page header can open a sidebar in the page footer; multiple triggers anywhere in the document can all point to the same panel.

The trigger action — data-ln-toggle-action

Default is toggle (flip to the opposite state). Override with data-ln-toggle-action="open" or "close" to force one direction — the canonical pattern for the X close button inside a sidebar drawer or dismissible alert.

What state goes where

"Is this panel open?" lives on the panel's data-ln-toggle attribute. "Is the trigger's chevron rotated?" lives on the trigger's aria-expanded (synced from panel state). "What animation runs?" lives in CSS reading the .open class. The toggle's instance state is two fields: dom and isOpen. The attribute is the source of truth; isOpen is a cached comparison the observer uses to skip no-op transitions.

Single source of truth

Every consumer that wants to change state writes the attribute. There is no imperative API method — setAttribute is the only path.

const el = document.getElementById('my-panel');

// Open / close / read state
el.setAttribute('data-ln-toggle', 'open');
el.setAttribute('data-ln-toggle', 'close');
el.lnToggle.isOpen;       // boolean — read-only
el.lnToggle.destroy();    // detach listeners, dispatch :destroyed

Trigger click, sibling component, external script, DevTools — all write the attribute. The MutationObserver runs the same pipeline regardless of caller: .open class addition, aria-expanded sync on every matching trigger, :open event dispatch, localStorage write. One channel. One audited path.

All events bubble. detail.target is always the panel element, not a trigger. When :before-open or :before-close is canceled, the observer reverts the attribute; the revert is itself an attribute change, but the no-op guard at the top of _syncAttribute catches it (since isOpen never updated). No infinite loop.

Example — Multiple triggers per panel

Triggers are matched by ID, not by DOM proximity. Multiple triggers pointing to the same panel all work; clicking any one toggles the panel; all of them have their aria-expanded synced. Useful when the same action is exposed from multiple parts of a layout — header bar, sidebar, inline content.

<button data-ln-toggle-for="info">Top trigger</button>
<button data-ln-toggle-for="info">Side trigger</button>

<section id="info" data-ln-toggle>
    <p>Either button toggles me; both stay in sync.</p>
    <button aria-label="Close"
            data-ln-toggle-for="info"
            data-ln-toggle-action="close">×</button>
</section>

The aria sync iterates document.querySelectorAll('[data-ln-toggle-for="info"]') on every state change, so any number of triggers at any DOM location stay coherent.

Persistence

Add data-ln-persist to a panel to remember its open/closed state in localStorage. State restores on page load before the first event fires. Storage key format: ln:toggle:{pagePath}:{id}.

MarkupPageStorage key
<aside id="sidebar" data-ln-toggle data-ln-persist>/admin/usersln:toggle:/admin/users:sidebar
<aside id="sidebar" data-ln-toggle data-ln-persist="left-panel">/admin/usersln:toggle:/admin/users:left-panel
<section id="faq-1" data-ln-toggle data-ln-persist>/ln:toggle:/:faq-1

Path is part of the key on purpose — same id="sidebar" on different pages resolves to two different keys, so each page remembers its own sidebar state. Restore happens inside the constructor before reactivity starts; the user's choice always wins over markup defaults on subsequent visits.

If data-ln-persist is present but the element has neither an id nor an explicit key value, the persist module logs a warning and silently skips — the toggle continues to function without persistence. localStorage failures (private browsing, quota exceeded) are also silent.

Cancelable transitions

Listeners on :before-open and :before-close can call preventDefault() to halt the transition. The observer reverts the attribute; the revert is itself an attribute change, but the no-op guard catches it. Use this for permission checks, unsaved-data warnings, async confirmations.

document.addEventListener('ln-toggle:before-open', function (e) {
    if (e.detail.target.id === 'admin-panel' && !window.currentUser.isAdmin) {
        e.preventDefault();
        // observer reverts attribute back to "close"
        // _syncAttribute re-fires but no-ops because isOpen never flipped
    }
});

Do not write data-ln-toggle on another panel from inside a :before-open handler. Cancelable handlers conflate "I might prevent this" with "I want to react to it" — listen to the post-event (:open) for fan-out instead.

Cross-component coordination

ln-toggle is the foundation; several other components are thin layers on top. They all reach the toggle the same way: listen for events, or write the attribute.

ComponentHow it uses ln-toggle
ln-accordionListens for bubbled ln-toggle:open on the wrapper. Sets data-ln-toggle="close" on every other open sibling. Each sibling's observer runs its own close pipeline. Never reaches into a sibling's instance — the attribute is the only path.
ln-dropdownListens for ln-toggle:open / :close on the inner menu. On open: teleport to <body>, position, attach outside-click and resize-close listeners. On close: reverse. Writes data-ln-toggle="close" from the outside-click and resize handlers.
ln-translationsThe language-add menu uses a [data-ln-dropdown] > [data-ln-toggle] shape. After the user picks a language, ln-translations writes data-ln-toggle="close" to dismiss the menu.
App shell sidebar@mixin sidebar-drawer reads [data-ln-toggle="open"] directly from CSS for the translate-x animation. The scrim overlay reacts via sibling selector. No JS coordinator.
Dismissible alert.alert[data-ln-toggle="close"] { display: none; } in scss/components/_alert.scss hides closed alerts. With data-ln-persist, the dismissal sticks.

ln-modal, ln-popover, and ln-tabs are NOT built on ln-toggle. They have their own state machines because they need different contracts (modal owns ESC + scrim + focus trap; tabs own active-panel selection; popovers own positioning). If a piece of UI feels open/close-shaped but ships with a different attribute surface, that's intentional.

Common mistakes

Setting data-ln-toggle on a panel without an id

The panel works (you can flip data-ln-toggle on it directly via setAttribute), but no [data-ln-toggle-for] trigger can find it — document.getElementById returns null and the click handler returns silently. Add an id.

Removing data-ln-toggle to "close" the panel

removeAttribute('data-ln-toggle') does NOT close the panel via the toggle pipeline. The .open class persists; aria-expanded does not update. To close, write the value: setAttribute('data-ln-toggle', 'close'). To dispose entirely, call destroy() first.

Putting data-ln-toggle on <main>

HTML spec allows only one <main> per page. Use <section>, <aside>, <article>, or <div> with a class.

Adding padding to .collapsible directly

The collapse animation transitions grid-template-rows from 0fr to 1fr. Padding on the parent shows as a thin always-visible strip when collapsed. Padding goes on .collapsible-body, never on .collapsible.

Expecting ESC or outside-click to close

ln-toggle has no keyboard handler and no document-level click listener. For ESC-to-close drawers, wire it on the consumer side (document.addEventListener('keydown', ...)). For dropdown-style outside-click, use ln-dropdown. For modal-style ESC + focus trap + scrim, use ln-modal (which is NOT built on ln-toggle).

Putting data-ln-toggle and data-ln-toggle-for on the same element

The element becomes a panel that controls itself — clicking it would call toggle() on its own instance. Rarely intentional. Split into trigger and panel.

Persisting a panel without id or explicit key

data-ln-persist in boolean form requires an id. Without one, the persist module logs [ln-persist] Element requires id or data-ln-persist="key" and silently skips. Use data-ln-persist="custom-key" if the panel has no id.

HTML structure reference

<!-- Panel that will be toggled -->
<aside id="my-sidebar" data-ln-toggle>
    Content
</aside>

<!-- Trigger that controls it -->
<button data-ln-toggle-for="my-sidebar">
    Toggle
</button>

<!-- Starts open -->
<section id="panel" data-ln-toggle="open">
    Open content
</section>

<!-- Explicit close-only action — for X buttons inside drawers/alerts -->
<button data-ln-toggle-for="panel"
        data-ln-toggle-action="close">
    Close
</button>

<!-- Persisted across reloads -->
<aside id="sidebar" data-ln-toggle data-ln-persist>
    State remembered in localStorage
</aside>

CSS for animations

ln-toggle adds/removes the .open class. Visual transition is the consumer's CSS. Three patterns are common:

// Sidebar — transform animation
.sidebar {
    transform: translateX(-100%);
    @include transition;
    &.open { transform: translateX(0); }
}

// Collapsible — grid-template-rows (RECOMMENDED for height animation)
<section id="panel" data-ln-toggle class="collapsible">
    <article class="collapsible-body">
        Content...
    </article>
</section>

// Or via @include in project SCSS:
.my-panel              { @include collapsible; }
.my-panel > .my-body   { @include collapsible-content; }

// Hidden via display:none (for dismissible alerts)
.alert[data-ln-toggle="close"] { display: none; }

The library ships @mixin collapsible (grid-template-rows pattern), @mixin sidebar-drawer (translate-x pattern), and the dismissible-alert pattern out of the box.

Dynamic DOM

The component uses a document-level MutationObserver — it automatically initializes new elements added to the DOM, including content from AJAX, innerHTML replacement, or template-driven rendering.

// No manual initialization needed.
// MutationObserver detects new [data-ln-toggle] elements.

fetch('/partial').then(r => r.text()).then(html => {
    document.getElementById('content').innerHTML = html;
    // ln-toggle automatically initializes the new elements
});

window.lnToggle(root) is escape-hatch only — for Shadow DOM, iframes, or any context the document-level observer doesn't see.

Practical uses

  • Sidebar drawer — pair with @mixin sidebar-drawer; persist with data-ln-persist; close button uses data-ln-toggle-action="close".
  • Inline collapsibles — pair with @mixin collapsible; chevron icon rotates via aria-expanded automatically.
  • Accordion panels — wrap with [data-ln-accordion] for single-open coordination. See the accordion docs.
  • Dropdown menus — wrap with [data-ln-dropdown] for outside-click, repositioning, and z-index teleport. See the dropdown docs.
  • Dismissible alerts — combine with data-ln-persist; .alert[data-ln-toggle="close"] hides the alert globally.
  • Conditional panels — listen for :before-open and call preventDefault() on permission failure.
  • URL-driven panels — listen for hashchange and write data-ln-toggle="open" on the panel matching the hash.
