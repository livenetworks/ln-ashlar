Overview & Architecture

For applications where fast initial paint is critical but huge background datasets (10,000+ rows) are needed for offline usage or responsive scrolling, ln-table provides a high-performance **SSR Hydration & Data Sync** pipeline.

Lifecycle Mechanics

1. **Instant SSR Paint**: The table loads with a fast pre-rendered subset of rows (e.g. 10 rows printed inside <tbody> by the server). 2. **Local Hydration**: The JS layer instantly parses these pre-rendered rows on load, enabling immediate client-side sort and search. 3. **Background Sync**: Simultaneously, the coordinator fetches the complete 10,000 records from the PHP backend API. 4. **Smooth Virtualization Upgrade**: When the API completes, the table merges the dataset, upgrades to Virtual Scroll, and updates page counters.

Remote Full-Text Search Integration

When searching, the table first filters local cached items for instant visual response, and then dispatches the search term to the PHP backend. The PHP API runs a full-text search across title, department, owner, and tags properties, returning matches from the entire 10k database.
