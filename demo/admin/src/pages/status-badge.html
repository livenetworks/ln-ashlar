Production Usage Pattern

In production, avoid class utilities like .success on project-owned elements. Use @include badge on a semantic selector and override --color-primary:

<!-- Library classes (prototyping / design system demos) -->
<span class="badge success">Active</span>
<span class="badge warning">Pending</span>
<span class="badge neutral">Inactive</span>

<!-- Production pattern — project-owned selectors -->
<span class="status active">Active</span>
<span class="status pending">Pending</span>
<span class="status inactive">Inactive</span>
// Project SCSS
.status          { @include badge; }
.status.active   { --color-primary: var(--color-success); }
.status.pending  { --color-primary: var(--color-warning); }
.status.inactive { --color-primary: var(--color-text-muted); }

// Add pulse for real-time states
.status.live     { @include badge-live; }
