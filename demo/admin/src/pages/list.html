Overview & Architecture

For applications where fast initial paint is critical but huge background datasets (10,000+ rows) are needed for offline usage or responsive scrolling, ln-list provides a high-performance **SSR Hydration & Data Sync** pipeline matching the ln-table synchronization protocol.

Lifecycle Mechanics

1. **Instant SSR Paint**: The list loads with a fast pre-rendered subset of items (e.g. 10 items printed inside the container by the server). 2. **Local Hydration**: The JS layer instantly parses these pre-rendered items on load, enabling immediate client-side search. 3. **Background Sync**: Simultaneously, the coordinator fetches the complete 10,000 records from the PHP backend API. 4. **Smooth Virtualization Upgrade**: When the API completes, the list merges the dataset, upgrades to Virtual Scroll, and updates page counters.

Remote Full-Text Search Integration

When searching, the list first filters local cached items for instant visual response, and then dispatches the search term to the PHP backend. The PHP API runs a full-text search across title, department, owner, and tags properties, returning matches from the entire 10k database.
