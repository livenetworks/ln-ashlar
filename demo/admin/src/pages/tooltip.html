Tooltip

Tooltip has two layers. CSS baseline via data-ln-tooltip="…" — zero JS, shows on :hover and :focus-visible. JS enhance via data-ln-tooltip-enhance — progressive enhancement over the same attribute, adds viewport-aware placement (auto-flip at edges), aria-describedby wiring, and portal-rendered nodes that escape overflow: hidden ancestors. Sections 1–3 below demo the baseline; sections 4–8 demo the enhance layer.

1. CSS baseline — default position (top)

Icon buttons with data-ln-tooltip="…". The default position is top. aria-label is still required for screen readers — the tooltip text is only visual.

2. CSS baseline — all four positions

Override the default with data-ln-tooltip-position="bottom|left|right". The compass layout below places one trigger per position.

3. CSS baseline — on inline text

Hover over ISO 9001 to see the full standard name. This is the pure CSS baseline — no JS runs for elements without a title attribute.

4. JS enhance — side by side

Hover both buttons. Both show a tooltip. Only the enhanced button gets aria-describedby when focused — inspect with DevTools.

Tab to the JS-enhanced button. Open DevTools Elements tab. The button should have aria-describedby="ln-tooltip-1" pointing at a node inside #ln-tooltip-portal.

5. JS enhance — edge auto-flip (right → left)

The last button is pushed to the far right edge of the container. Its preferred side is right. Hover it — the tooltip should appear on the left because there is no room on the right. Inspect data-ln-tooltip-placement on the tooltip node.

6. JS enhance — long-text max-width wrap

The tooltip below has text that exceeds 20 rem. Hover it to verify the max-width: 20rem wraps the text instead of overflowing the viewport.

7. JS enhance — keyboard focus + aria-describedby

Instructions: Tab to the button below. Open DevTools (F12) and switch to the Elements panel. Inspect the button — it should have aria-describedby="ln-tooltip-N". Inspect #ln-tooltip-portal in the body — there should be a <div class="ln-tooltip"> with the matching id. Tab away — both attributes should disappear.

8. JS enhance — auto-attach on [data-ln-tooltip][title]

Any element with both data-ln-tooltip and title auto-enhances — the JS layer attaches without requiring data-ln-tooltip-enhance. The CSS baseline cannot strip title, so the browser's native tooltip would leak alongside the styled one. Auto-enhance resolves this for the semantic <abbr> pattern.

Hover (or tab to) ISO. Open DevTools and watch the title attribute — it is stripped while the styled tooltip is visible and restored on hide. No data-ln-tooltip-enhance flag is required.

9. SCSS usage

// CSS baseline — attribute selector, no config needed
// <button data-ln-tooltip="Save" aria-label="Save">...</button>

// Positions
// data-ln-tooltip-position="top"    (default)
// data-ln-tooltip-position="bottom"
// data-ln-tooltip-position="left"
// data-ln-tooltip-position="right"

// JS enhance — add data-ln-tooltip-enhance for viewport-aware
// placement + aria-describedby wiring
// <button data-ln-tooltip="Save" data-ln-tooltip-enhance>...</button>

// Consumer semantic selector for the baseline mixin
.my-trigger { @include tooltip; }
