Overview & Philosophy

A dropdown menu is one of the most common layout controls in web applications. While simple on the surface, a premium dropdown menu requires complex engineering: it must close when the user clicks outside, close on viewport resizes, teleport to the body to escape parent stacking contexts (preventing overflow: hidden clippings), and position itself dynamically to fit the viewport.

In ln-ashlar, we solve this elegantly. ln-dropdown is a coordinator wrapper that layers this menu-grade behavior on top of a standard ln-toggle instance. The dropdown component does not re-implement state; it delegates 100% of its open/closed state management to the underlying toggle primitive, adding click-outside listeners and positioning math on top.

The Dropdown Doctrine

A dropdown does not own its state. The open/closed state lives entirely in the data-ln-toggle attribute of the inner menu. Click-outside rules and resize handlers close the menu simply by writing data-ln-toggle="close" back to the menu. The observer on the toggle runs the close pipeline, which the dropdown coordinates. The HTML attribute is the sole contract.

Layout Teleportation & Positioning

When a dropdown menu is activated, ln-dropdown teleports the menu element to the <body> element to prevent clipping under scrollable containers. It then measures the trigger element and matches its position right-aligned below the trigger. It automatically flips above if vertical space is restricted, or left-aligned if horizontal space is restricted, returning the menu back to its original DOM position when closed.

For the complete developer reference, see the ln-dropdown README and the architecture reference.
