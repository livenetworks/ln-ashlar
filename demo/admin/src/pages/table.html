Table

Server-rendered data table — in-memory search, per-column filtering, sorting, virtual scroll. Rows arrive as HTML in <tbody>. Two independent components: ln-table (parse + render) and ln-table-sort (sort header clicks). Search provided by ln-search.

Attributes

AttributeOn elementDescription
data-ln-tablewrapper (needs id)Component root. Parses rows, listens for events.
data-ln-search="tableId"<input> or wrapperStandard ln-search. ln-table intercepts the event for in-memory filtering.
data-ln-table-clear<button>Clears search input.
th[data-ln-sort="type"]<th>Sortable column. Values: string, number, date.
td[data-ln-value]<td>Raw sort value when display text differs.
data-ln-filter-col="key"<th>Maps column to a filter key (used by ln-filter).
data-ln-table-empty<template>Content shown when filter returns 0 results.

Events

EventDispatched bydetail
ln-table:readyln-table — after row parse{ total }
ln-table:filterln-table — after filter render{ term, matched, total }
ln-table:sortln-table-sort — on th click{ column, sortType, direction }
ln-table:sortedln-table — after sort render{ column, direction, matched, total }
ln-table:emptyln-table — filter returns 0{ term, total }
 
var table = document.getElementById('demo-table');

table.addEventListener('ln-table:ready', function (e) {
    countEl.textContent = e.detail.total.toLocaleString() + ' products';
});

table.addEventListener('ln-table:filter', function (e) {
    countEl.textContent = e.detail.matched < e.detail.total
        ? e.detail.matched.toLocaleString() + ' / ' + e.detail.total.toLocaleString()
        : e.detail.total.toLocaleString() + ' products';
});

data-ln-value — sort raw value

When display text differs from the sort key:

<!-- Without: sorts by "1.250" (string) -->
<td>1.250</td>

<!-- With: sorts by 1250 (number) -->
<td data-ln-value="1250">1.250</td>

<!-- Date: sorts by Unix timestamp -->
<td data-ln-value="1700000000">15.11.2023</td>

Virtual Scroll

Activates when filtered rows exceed 200. Renders only visible rows + 15 buffer rows above/below. Deactivates automatically when rows drop below threshold. Column widths are locked on first parse to prevent layout shifts.

Dynamic Rows (AJAX)

If <tbody> is empty at init, a MutationObserver waits for rows to appear and parses them automatically.

// Set innerHTML directly — ln-table detects it
document.querySelector('#products tbody').innerHTML = generatedHtml;

HTML Structure

<div id="products" data-ln-table>
    <header class="ln-table__toolbar">
        <h3>Products</h3>
        <aside>
            <label>
                <span class="ln-icon-search ln-icon--sm"></span>
                <input type="search" placeholder="Search..." data-ln-search="products">
            </label>
        </aside>
    </header>

    <template data-ln-table-empty>
        <article class="ln-table__empty-state">
            <h3>No results</h3>
            <button data-ln-table-clear>Clear</button>
        </article>
    </template>

    <table>
        <thead>
            <tr>
                <th data-ln-sort="number">#</th>
                <th data-ln-sort="string">Name</th>
                <th data-ln-sort="date">Date</th>
                <th data-ln-sort="number">Price</th>
                <th>Actions</th>
            </tr>
        </thead>
        <tbody>
            <tr>
                <td>1</td>
                <td>Widget A</td>
                <td data-ln-value="1700000000">15.11.2023</td>
                <td data-ln-value="1250">1.250</td>
                <td><button>Edit</button></td>
            </tr>
        </tbody>
    </table>

    <footer class="ln-table__footer">
        <span>Total: <strong></strong></span>
    </footer>
</div>

API

// Manual init
window.lnTable(document.body);

// Instance
var el = document.querySelector('[data-ln-table]');
el.lnTable.destroy();   // remove listeners, virtual scroll, column locks
