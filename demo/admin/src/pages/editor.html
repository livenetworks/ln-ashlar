Overview & Philosophy

ln-editor is a lightweight WYSIWYG rich text editor that follows ln-ashlar's progressive enhancement doctrine. It enhances a standard <textarea> into a contentEditable editing surface with a toolbar. Without JavaScript, the textarea works as a plain text input — zero functionality is lost.

The toolbar is authored HTML, not generated by JS. The consumer controls which formatting buttons appear, their order, and their labels. This follows Mindset §1: "Markup Is the Application." The editing surface uses @include prose so content looks identical to how it will render in a .prose display container.

How it works

  1. JS finds the <textarea> inside [data-ln-editor]
  2. Creates a <div contenteditable> editing surface
  3. Hides the textarea (CSS, via data-ln-editor-source)
  4. On every content change, syncs surface.innerHTML → textarea.value
  5. On form submit, the textarea already has the current HTML — the server receives it

For full documentation see the ln-editor README and the architecture reference.

Anatomy

The minimal markup contract:

<div data-ln-editor>
    <nav>
        <ul>
            <li><button type="button" data-ln-editor-action="bold" aria-label="Bold">
                <svg class="ln-icon" aria-hidden="true"><use href="#ln-bold"></use></svg>
            </button></li>
            <li><button type="button" data-ln-editor-action="italic" aria-label="Italic">
                <svg class="ln-icon" aria-hidden="true"><use href="#ln-italic"></use></svg>
            </button></li>
        </ul>
    </nav>
    <textarea name="content" placeholder="Write something..."></textarea>
</div>

Toolbar structure

<nav> wraps the toolbar. Each <ul> is a visual button group — multiple <ul> elements create separator lines between groups. <button data-ln-editor-action="..."> binds each formatting action. The consumer decides which actions to include.

Attributes reference

AttributeOn elementOwned byDescription
data-ln-editorContainer <div>ln-editorMarks the container as an editor instance
data-ln-editor-action<button>ln-editorFormatting action to execute on click
data-ln-editor-source<textarea>ln-editor (auto)JS sets this to hide the textarea via CSS. Removed on destroy()

Progressive enhancement

Without JS, the <nav> toolbar is inert and the <textarea> is a normal text input. With JS, the textarea is hidden and the contentEditable surface takes over.

Example — Full toolbar

All available formatting actions. Active buttons are highlighted with the accent color as you move the cursor through formatted content.

Example — Minimal toolbar

Only bold, italic, and link — suitable for comments or short descriptions. The toolbar is authored HTML — include only what you need.

Example — Pre-filled content

Server-rendered HTML in the textarea is used as initial editor content. The editing surface renders it with @include prose styling.

Example — Form integration

The editor syncs content to the hidden <textarea> on every change. Click Save to see the raw HTML that would be submitted.

API

						


const el = document.querySelector('[data-ln-editor]');

// Read current HTML content
el.lnEditor.getHTML();

// Set content programmatically
el.lnEditor.setHTML('<p>New content</p>');

// Or via request event (coordinator pattern)
el.dispatchEvent(new CustomEvent('ln-editor:set-content', {
    detail: { html: '<p>Content via event</p>' }
}));

// Destroy — restores textarea visibility
el.lnEditor.destroy();
