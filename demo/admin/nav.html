    Nav

    Active link highlighter — automatically marks the active link in navigation based on the current URL. Works with pushState (AJAX) and popstate (browser back/forward).

    Basics

    AttributeOn elementValueDescription
    data-ln-nav<nav>CSS classActivates nav highlighter

    Example: data-ln-nav="active" — the link receives class .active when it is active.

    Behavior and Match Rules

    How does it work?

    1. On init, it scans all <a> elements inside <nav>
    2. It compares the href of each link with window.location.pathname
    3. Adds a CSS class to matching links

    Match Rules

    RuleExampleResult
    Exact match/users on /users✓ Match
    Parent match/users on /users/123✓ Match
    Root special/ only on /✓ Exact only
    Trailing slash/users/ == /users✓ Normalized

    HTML Example

    <nav data-ln-nav="active">
    <a href="/">Home</a>
    <a href="/users">Users</a>
    <a href="/users/new">New User</a>
    <a href="/settings">Settings</a>
</nav>

<!-- If the URL is /users/42 -->
<!-- The /users link receives class .active (parent match) -->
<!-- The /users/new link does NOT receive the class -->

    CSS Styling

    nav a {
    @include text-secondary;
    @include transition;

    &.active {
        @include text-primary;
        @include font-bold;
        @include bg-secondary;
        @include px(0.75rem);
        @include rounded-md;
    }
}

    Reactivity with AJAX and Browser History

    pushState (AJAX navigation)

    When ln-ajax changes the URL via pushState, ln-nav automatically updates the active classes.

    popstate (browser back/forward)

    On browser back/forward, ln-nav listens to the popstate event and updates the active classes.

    Dynamically added links

    New links added via JavaScript (MutationObserver) are automatically processed.

    API

    // Manual initialization
window.lnNav.init(navElement);

// Manually update active links
// (usually not needed — works automatically)
window.lnNav.updateActive();
