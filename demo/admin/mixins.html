    Mixins Reference

    Complete list of @include mixins from scss/config/_mixins.scss. Import with @use 'config/mixins' as *; and use in semantic selectors.

    Two-Layer Architecture: Mixins + Components

    Every visual style has two layers: a mixin (recipe) and a component (default application).

    Mixin (recipe)

    // scss/config/mixins/_table.scss
@mixin table-base {
  @include w-full;
  @include bg-primary;
  @include rounded-lg;
  // ... styles
}

    Component (default)

    // scss/components/_tables.scss
table {
  @include table-base;
}

@media (max-width: 768px) {
  table { @include table-responsive; }
}

    When to use which selector

    SituationMixinComponent selector
    Universal element (label, table, input)yestable { @include table-base; }
    Singleton (#breadcrumbs)yes#breadcrumbs { @include breadcrumbs; }
    Component class (.btn, .collapsible)yes.btn { @include btn; }
    Data-attr JS ([data-ln-tabs])not needed[data-ln-tabs] nav { ... }

    Project overrides — 4 levels

    NeedApproachExample
    Use default Do nothing — library CSS works <table>...</table>
    Different color Override CSS variable on parent #danger-zone { --color-primary: var(--color-error); }
    Different structure Re-apply mixin with modifications #audit-log { @include table-base; @include table-striped; }
    Full replace Use only the mixin on custom selector Exclude component, write from scratch

    Override Architecture — Zero Hardcoded Colors

    Mixins NEVER contain concrete color values (#fff, rgb(), hsl(0 0% 100%)). Every color reads from a CSS custom property token: var(--color-*). This is what makes the entire system work — colors are inherited through the CSS cascade, not baked into mixin definitions.

    WRONG

    // Hardcoded — breaks inheritance
.ln-modal header {
  background-color: hsl(var(--color-primary));
  border-bottom: none;
  color: #fff;
}

    CORRECT

    // Mixin reads tokens, project just uses it
.ln-modal header {
  @include panel-header;
}

// Want a different color? Override the token
.ln-modal {
  --color-primary: 280 60% 50%;
  // header automatically inherits
}

    When to use which approach

    NeedApproachExample
    Different color Override CSS variable on parent element .ln-modal { --color-primary: 280 60% 50%; }
    Different structure (padding, layout, shape) Override the mixin in project _overrides.scss @mixin panel-header { @include px(2rem); ... }
    Both Override mixin for structure + CSS variable for color Mixin override + .parent { --color-primary: ...; }

    ln-acme default

@mixin btn {
  @include flex;
  @include items-center;

  @include py(0.5rem);
  @include text-sm;
  @include rounded-md;
  @include cursor-pointer;
  @include transition;
  border: none;
  outline: none;
  white-space: nowrap;
}

    Project override

    // project/scss/_overrides.scss
@mixin btn {
  @include inline-flex;
  @include items-center;
  @include px(1.5rem);
  @include py(0.75rem);
  @include text-base;      // larger text
  @include font-medium;
  white-space: nowrap;
}

// Project selectors — UNCHANGED
#save-user   { @include btn; }
#delete-user { @include btn; }

    How to set up the override

    // project/scss/main.scss
// 1. Import ln-acme (loads default mixins)
@use 'ln-acme/scss/ln-acme';

// 2. Import project overrides AFTER ln-acme
//    This file redefines mixins with the same name
@use 'overrides';

// 3. Import project pages (use the mixins — they get the overridden version)
@use 'pages/users';
@use 'pages/dashboard';

    Container Queries

    Parent declares the container context, child queries it with native @container. Never combine container-type with overflow: hidden on the same element.

    MixinOutput
    @include containercontainer-type: inline-size
    @include container(name)container-type: inline-size; container-name: name
     
    // Parent — declares the context
#sidebar { @include container(sidebar); }

// Child — queries the container
#sidebar nav {
  @container sidebar (min-width: 580px) {
    display: grid;
    grid-template-columns: repeat(2, 1fr);
  }
}

    Collapsible

    Grid-based expand/collapse animation. Uses grid-template-rows: 0fr/1fr transition.

    MixinRole
    @include collapsibleParent — display: grid with 0fr → 1fr transition. Toggled by .open class
    @include collapsible-contentChild — overflow: hidden to clip content
     
    .my-panel          { @include collapsible; }
.my-panel > section { @include collapsible-content; }

    Buttons

    Two-tier system: structure is global, color is semantic.

    MixinDescription
    @include btnComplete action button: structure (inline-flex, padding, font) + filled primary color. For non-submit action buttons via project SCSS.
    @include btn-smSize modifier: less padding, text-xs. Apply after @include btn.
    @include btn-lgSize modifier: more padding, text-base. Apply after @include btn.
    @include btn-groupRow action button group: inline-flex, small gap, no list style.
    @include close-buttonIcon close button: resets to padding: 0, fixed 2rem size, hover-red color.

    Global button tiers

    TierHowResult
    1. Neutral<button type="button"> — no classTransparent bg, muted text, gray hover. Cancel/Close/Toggle.
    2. Primary<button type="submit"> — no classPrimary fill color + white text. Color-only override on top of tier 1 structure.
    3. Action@include btn on a semantic selector (or class="btn" for prototyping)Filled primary color. Same as tier 2 but for non-submit action buttons.

    Live demo

    Color change — override --color-primary

    Set --color-primary on the element or any parent. All states (hover, active, focus-ring) adapt automatically.

    /* Project semantic selectors — correct approach */
#delete-user  { @include btn; --color-primary: var(--color-error); }
#approve-user { @include btn; --color-primary: var(--color-success); }

/* Context override — all submit + action buttons inside change */
.danger-zone  { --color-primary: var(--color-error); }

/* Custom brand color */
:root { --color-brand: 280 60% 50%; }
.brand-section { --color-primary: var(--color-brand); }
