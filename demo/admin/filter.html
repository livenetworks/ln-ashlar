    Filter

    Generic filter component — filters children of a target element by data-* attribute. Supports multi-select: multiple checkboxes can be active simultaneously (OR logic). Works independently alongside ln-search.

    Attributes

    AttributeOn elementDescription
    data-ln-filter="targetId"component rootTarget element by ID whose children are filtered
    data-ln-filter-key="field"<input type="checkbox">Name of data attribute for comparison on target children
    data-ln-filter-value="val"<input type="checkbox">Value for comparison
    data-ln-filter-reset<input type="checkbox">Marks the "All" (reset) checkbox. Replaces data-ln-filter-value="" which still works as fallback.
    data-ln-filter-hidetarget childrenSet by JS when element doesn't match
    data-ln-filter-colComponent rootColumn index (0-based) for table row filtering. Reads <td> text at column N.

    HTML Structure

    <!-- Filter checkboxes -->
<nav data-ln-filter="my-list">
    <ul>
        <li><label><input type="checkbox" data-ln-filter-key="category" data-ln-filter-reset checked> All</label></li>
        <li><label><input type="checkbox" data-ln-filter-key="category" data-ln-filter-value="a"> Category A</label></li>
        <li><label><input type="checkbox" data-ln-filter-key="category" data-ln-filter-value="b"> Category B</label></li>
    </ul>
</nav>

<!-- Target list (children have data-category attribute) -->
<ul id="my-list">
    <li data-category="a">Element from category A</li>
    <li data-category="b">Element from category B</li>
</ul>

    Required CSS

    The hide rule is bundled in ln-acme. Pill styling (active state highlight) is automatic via label:has(> input:checked) in the library defaults — no additional CSS needed for active state.

    /* Combined with ln-search */
[data-ln-search-hide],
[data-ln-filter-hide] {
    display: none;
}
