    Persist

    localStorage persistence for UI component states. Opt-in via data-ln-persist. State is page-scoped — storage keys include the URL pathname so the same id on different pages never collides.

    How to test: Change any state below, then refresh the page — the state should be restored exactly as you left it.

    1 — Toggle persistence

    Open/close these sections, then refresh. The two persisted toggles restore; the control toggle always resets to closed.

    Persisted toggle A id="persist-toggle-a" data-ln-persist

    This section remembers whether it was open or closed. Open it and refresh the page — it will still be open.

    Storage key: ln:toggle:{path}:persist-toggle-a

    Persisted toggle B id="persist-toggle-b" data-ln-persist

    Independent from Toggle A. Both persist their own state separately.

    Storage key: ln:toggle:{path}:persist-toggle-b

    Control — no persistence id="no-persist-toggle" — no data-ln-persist

    This toggle has no data-ln-persist — it always resets to closed on page refresh. Nothing is written to localStorage.

    2 — Accordion persistence

    Open one panel and refresh. The same panel reopens — single-open constraint still works after restore.

    • Panel 1 — General settings

      Content of panel 1. Opening this closes panels 2 and 3.

      Storage key: ln:toggle:{path}:acc-panel-1

    • Panel 2 — Security

      Content of panel 2. Opening this closes panels 1 and 3.

      Storage key: ln:toggle:{path}:acc-panel-2

    • Panel 3 — Notifications

      Content of panel 3. Opening this closes panels 1 and 2.

      Storage key: ln:toggle:{path}:acc-panel-3

    3 — Tabs persistence

    Switch tabs, refresh. The non-hash tab set restores via localStorage. The hash tab set uses the URL — note the difference.

    Non-hash tabs — persists via localStorage

    data-ln-persist="persist-settings-tabs" — no id, no hash sync.

    General

    Switch to Security or Notifications, then refresh — that tab will be restored from localStorage.

    Storage key: ln:tabs:{path}:persist-settings-tabs

    Security

    This tab is active. Refresh the page — Security will still be active.

    Notifications

    This tab is active. Refresh the page — Notifications will still be active.

    Hash tabs — URL sync takes precedence

    id="hash-tabs-demo" — hash sync is active. data-ln-persist would be ignored even if added.

    Alpha

    This tab set uses URL hash for state: #hash-tabs-demo:beta. No localStorage is written.

    Beta

    Active tab is visible in the URL bar. Refresh — hash restores it, not localStorage.

    Gamma

    Hash format: #hash-tabs-demo:gamma

    4 — Table sort persistence

    Click a column header to sort, then refresh. The sort column and direction are restored.

    data-ln-persist goes on the [data-ln-table] wrapper — one attribute covers both ln-table and ln-table-sort.

    Name Department Role Salary Start date
    Alice Johnson Engineering Lead 95,000 01.01.2021
    Bob Martinez Design Senior 82,000 01.07.2021
    Carol White Engineering Mid 74,000 01.01.2022
    David Lee Marketing Senior 78,000 01.07.2022

    Storage key: ln:table-sort:{path}:persist-table — value: { col: N, dir: "asc"|"desc" }

    5 — Filter persistence

    Check one or more filters, then refresh. The selected values are restored and the list is filtered.

    • Alice Johnson — Active
    • Bob Martinez — Pending
    • Carol White — Active
    • David Lee — Archived
    • Eva Novak — Pending
    • Frank Okafor — Active
    • Grace Park — Archived

    Storage key: ln:filter:{path}:persist-filter — value: { key: "status", values: ["active"] }

    What each component stores

    ComponentKey patternExample value
    ln-toggleln:toggle:{path}:{id}"open" or "close"
    ln-tabsln:tabs:{path}:{key}"security"
    ln-table-sortln:table-sort:{path}:{id}{"col":3,"dir":"desc"}
    ln-filterln:filter:{path}:{id}{"key":"status","values":["active"]}
