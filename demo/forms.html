Forms & Buttons

CSS Grid + wrapping <label> pattern. No <div class="form-group"> or <div class="form-row">. The grid controls the layout via SCSS — not via HTML.

Principle

Each field is a <label> that wraps the text and the input (implicit association — no for/id). The root element (<form> or <section>) gets @include form-grid in SCSS — a 6-column grid, collapsing to 1 column on mobile. Column spans are set on > label:nth-child(N) in app-specific SCSS — never inline style="".

Wrong — old pattern

<!-- div wrappers, explicit for/id -->
<div class="form-group">
  <label for="name">Name</label>
  <input type="text" id="name">
</div>

<div class="form-row">
  <div class="form-group">
    <label for="city">City</label>
    <input type="text" id="city">
  </div>
  <div class="form-group">
    <label for="zip">Zip</label>
    <input type="text" id="zip">
  </div>
</div>

Correct — new pattern

<!-- label wrap, grid in SCSS -->
<form id="my-form">

  <label>
    Name
    <input type="text" name="name">
  </label>

  <label>
    City
    <input type="text" name="city">
  </label>

  <label>
    Zip
    <input type="text" name="zip">
  </label>

</form>
Element Rule
Root <form id="..."> — styled with @include form-grid in SCSS (6 cols → 1 col on mobile)
Children Direct <label> — no <div class="form-group"> or <div class="form-row">
Label Wraps the text + input (implicit association — no need for for/id). Checkbox/radio: text goes after the input
Column spans In SCSS via > label:nth-child(N) { grid-column: span N; } — NEVER inline style="". span 3 = ½, span 2 = ⅓, span 6 = full
Errors <small class="text-error"> inside <label> + small { display: block } in SCSS
Checkbox / Radio input[type="checkbox"] { width: auto } — ln-acme sets input { width: 100% } globally
<fieldset> ONLY for semantically related fields (e.g. address: street + city + postal code). NOT for visual rows
.form-actions Component class — stays in HTML. Gets grid-column: span 6 (full-width) in SCSS

Implementation

HTML

<form id="my-form">

  <!-- Standard field — span 3 (half) via SCSS -->
  <label>
    Name <span class="text-error">*</span>
    <input type="text" name="fname" required placeholder="...">
    <small class="text-error">Required field</small>
  </label>

  <!-- Select — same pattern -->
  <label>
    Category
    <select name="category">
      <option value="a">Option A</option>
    </select>
  </label>

  <!-- Checkbox — text AFTER the input -->
  <label>
    <input type="checkbox" name="confirmed" value="1">
    I confirm the data
  </label>

  <!-- form-actions is a component class, stays in HTML -->
  <div class="form-actions">
    <button type="button" class="btn btn--secondary">Cancel</button>
    <button type="submit" class="btn">Save</button>
  </div>

</form>

SCSS

#my-form {
  @include form-grid;       // 6 cols, gap, responsive (<768px → 1 col)

  > label {
    grid-column: span 3;    // default: half width
  }

  // Specific spans
  > label:nth-child(3) { grid-column: span 4; }   // 2/3 width
  > label:nth-child(4) { grid-column: span 2; }   // 1/3 width

  // Three in a row (span 2 × 3 = 6 cols)
  > label:nth-child(5),
  > label:nth-child(6),
  > label:nth-child(7) { grid-column: span 2; }

  // Full-width
  > label:nth-child(8),
  > label:nth-child(10),
  > .form-actions { grid-column: span 6; }

  // Checkbox/radio fix
  input[type="checkbox"],
  input[type="radio"] { width: auto; }

  // Error display
  small { display: block; }
}
